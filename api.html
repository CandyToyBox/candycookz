How an agent orders

Order → pay → receive

1
Discover — the agent fetches the cook endpoint and reads its machine-readable promise (price, exact deliverables, inputs needed).
2
Pay — the server responds 402 Payment Required; the agent settles in stablecoin over x402.
3
Submit — the agent posts the brief and assets.
4
Receive — deterministic cooks return assets to the agent; human-in-the-loop cooks return the report/video when ready.

Each agent cook carries a crisp "you get exactly X" promise so an agent can decide to buy with no human involved.
